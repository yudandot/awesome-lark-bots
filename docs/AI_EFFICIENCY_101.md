## 开场：先别慌

AI 相关的消息每天都在刷屏。GPT 又升级了、Sora 又出新版了、DeepSeek 又开源了、某某岗位要被取代了……

**先说一个结论：你不需要学会任何 AI 技术。**

真的。AI 技术迭代速度太快了——你今天花一周学的工具，下个月可能就被新的替代了。所以"学不学得会技术"不是重点。

重点是什么？

**是判断力。**

我们每天要判断的事情太多了——这个方向对不对、这个文案打不打动人、该坚持还是该转向。而每个人每天的判断力是有限的。如果大量精力消耗在了重复解释背景、反复对齐信息、开了两次会还没理清方向上，留给真正重要的决策的判断力就不够了。

AI 不是来取代我们的。它是来帮我们**把重复的、机械的部分接过去，让我们的判断力留给只有人能做的那些决策。**

这是今天分享的核心。后面所有的工具和技巧，都是围绕这一件事。

先看看 AI 现在到底能干嘛，你自己感受一下。

## 第一部分：AI 现在到底能干嘛

### 问答——大多数人停在这里

```
你 → "帮我写一段关于春天的文案"
AI → 给你一段文案
你 → 复制粘贴（或者觉得不太行，再问一遍）
```

一问一答，用完即走。AI 像一个更聪明的搜索引擎。

这是最基础的用法，没问题，但还有更多。

### 对话——进了一步

```
你 → "我们是一个咖啡品牌，目标是 25-35 岁城市白领..."
你 → "帮我想 5 个春季营销的选题方向"
AI → 给你 5 个方向
你 → "第 3 个不错，但要更强调线下体验"
AI → 修改后的版本
你 → "再帮我写成小红书文案"
AI → 给你文案
```

多轮对话，AI 开始理解你的上下文，结果质量明显好一些了。

但还是你在一步步带着 AI 走。换一个选题，又要从头来。

### 工作流——一句话触发一条产线

```
你 → "春天穿搭分享"（就这一句话）
AI → 自动扫描当前热点
   → 结合你的品牌调性产出 5 个创意
   → 为每个平台生成适配文案
   → 生成配图 Prompt
   → AI 自评质量
   → 存入仓库等你审批
```

你只说了一句话。剩下的，AI 自己编排、自己执行、自己检查。

**区别在哪？**
- 问答和对话：你是操作员，AI 是工具
- 工作流：你是决策者，AI 是你的执行团队

### 类比一下

| | 自己来 | 问 AI | 和 AI 聊 | AI 工作流 |
|---|---------|---------|---------|---------|
| **做菜** | 买菜洗菜切菜炒菜 | 问 AI 菜谱 | 跟 AI 讨论口味后定制菜谱 | 说"今晚想吃川菜"，AI 查冰箱、出菜谱、列采购清单 |
| **做内容** | 选题写稿改稿做图发布 | 让 AI 写一段文案 | 多轮讨论打磨文案 | 说"咖啡 × 音乐节"，AI 脑暴→文案→配图→适配各平台 |

## 第二部分：几个马上能用的技巧

不需要装任何东西。你手边的 ChatGPT、DeepSeek、Kimi、豆包，今天就能用。

### 技巧 1：先拆后做

别一上来就甩大任务。

```
❌ "帮我做一个完整的营销方案"
✅ "先帮我分析目标受众"
   → "再帮我想 5 个选题方向"
   → "选第 3 个，帮我写小红书文案"
```

AI 在小而明确的任务上表现远好于大而模糊的任务。就像你给同事布置工作一样——「帮我做个方案」和「帮我调研一下竞品的定价策略」，后者的结果一定更好。

### 技巧 2：告诉 AI 什么不该做

你告诉 AI「写一段品牌文案」，它大概率会给你一段充满「赋能」「沉浸式」「焕新」的东西。

```
✅ "帮我写一段品牌文案。
    禁止用'赋能'、'沉浸式'、'焕新'等营销黑话。
    禁止超过 200 字。
    语气像朋友推荐，不像广告。"
```

**限制产生质量。** 告诉 AI 不该做什么，比告诉它该做什么更有效。

### 技巧 3：给 AI 一个角色

AI 默认是"好好先生"——你说啥它都觉得好。你要它真的有用，就给它一个有态度的角色。

```
❌ "帮我优化这个方案"
✅ "假设你是一个严苛的投资人，这个方案最大的三个漏洞是什么？"
```

### 技巧 4：让两个 AI 互相挑战

一个 AI 生成，另一个 AI 挑毛病。

做法很简单：让 AI 生成内容后，开一个新对话，把内容粘进去，说「按这几个标准打分，不合格的说理由」：

```
评分标准：
1. 前 3 秒能否让人停下来？
2. 是否提供了真正的信息/情感/娱乐价值？
3. 看完后有没有分享冲动？
4. 是否符合品牌调性？
每项 0-1 分，低于 0.7 的不合格。
```

你会发现 AI 能帮你过滤掉很多「听起来不错但其实不行」的内容。

### 技巧 5：积累你的"AI 知识包"

每个人都有反复给 AI 解释的背景信息：公司做什么的、品牌调性、目标受众……

**做法：**
1. 建一个文档，把这些信息写清楚
2. 每次用 AI 前，先把这个文档喂给它
3. 发现 AI 误解什么就补一条

这就是「知识库」的土办法。不需要任何技术，效果立竿见影。

## 第三部分：我们做了一组飞书机器人

上面这些技巧手动操作完全可以，但每次都要操作一遍挺麻烦的。

所以我们把这些做成了飞书机器人——在飞书里发一句话就行，剩下的自动跑。

### 有哪些机器人

| 你想干嘛 | 用哪个 | 怎么用 |
|---------|--------|--------|
| 想选题、想创意 | 脑暴机器人 | 发一句话，5 个 AI 角色帮你吵出落地方案 |
| 做计划、做分析 | 规划机器人 | 发一句话，结构化产出执行计划 |
| 从选题到发布一条龙 | 自媒体助手 | 发一句话，自动跑完选题→文案→配图全流程 |
| 做视觉素材 | 创意 Prompt | 发一句话，生成可直接用的 AI 绘图 Prompt |
| 看品牌舆情 | 舆情监控 | 发一句话，15 个平台数据采集+分析 |
| 看行业新闻 | 早知天下事 | 每天自动推送新闻简报 |
| 备忘/日程/杂事 | 助手机器人 | 说人话就行，它自己分类处理 |

### 背后用了什么 AI

不同任务用不同模型，就像团队里不同的人擅长不同的事：

| 需要什么 | 选什么模型 |
|---------|-----------|
| 逻辑推理、结构化分析 | DeepSeek、Claude |
| 创意发散、文案写作 | 豆包、GPT-4 |
| 长文本理解、资料整理 | Kimi、Claude |
| 快速问答、轻量任务 | DeepSeek |

不绑定任何一家。DeepSeek、GPT、Claude、豆包、Kimi 都可以用，随时可以换。

### 4.1 安装（一次性，20 分钟）

> 完整的零基础安装教程见 [SETUP_GUIDE.md](SETUP_GUIDE.md)，以下是快速版。

**你需要三样东西：**

| 要什么 | 去哪拿 | 干什么用 |
|-------|--------|---------|
| 飞书 App ID + Secret | [飞书开放平台](https://open.feishu.cn) → 创建应用 | 让机器人连上飞书 |
| DeepSeek API Key | [platform.deepseek.com](https://platform.deepseek.com) → API Keys | 主力 AI 模型 |
| 豆包 + Kimi API Key | 火山控制台 / Moonshot 平台 | 脑暴专用（可选） |

> 只想先试试？只需要飞书 + DeepSeek 两个就够了。

**四步启动：**

```bash
# 1. 下载代码
git clone <项目地址> && cd AIlarkteam

# 2. 装依赖（一行命令）
pip3 install -r requirements.txt

# 3. 配置（复制模板，填三个值）
cp .env.example .env
# 编辑 .env，填入 FEISHU_APP_ID、FEISHU_APP_SECRET、DEEPSEEK_API_KEY

# 4. 启动！
python3 -m brainstorm      # 脑暴机器人
# 或者 planner / assistant / creative / sentiment / newsbot / conductor
```

看到终端打印「正在连接飞书…」且没有报错，就是成功了。

**飞书端配置要点（详细步骤见 [SETUP_GUIDE.md](SETUP_GUIDE.md)）：**
1. 飞书开放平台创建「企业自建应用」
2. 启用「机器人」能力
3. 事件订阅选「长连接」模式（不需要公网 IP）
4. 添加事件 `im.message.receive_v1`
5. 发布应用

> 遇到问题？排查指南见 [SETUP_GUIDE.md → 出了问题？](SETUP_GUIDE.md#出了问题)

**费用：** 项目本身免费。AI API 按量付费——DeepSeek 约 ¥0.01/次对话，一次完整脑暴约 ¥0.2-0.5。各平台新注册通常都有免费额度。

## 第五部分：为什么我们要做这个

开头说了，核心是判断力。这部分展开聊聊。

### AI 很强，但各干各的不行

每个人身边都多了一个很能干的 AI 助手——但如果各干各的、信息不通：
- 你不知道她上午已经脑暴过同一个选题了
- 她不知道你昨天已经决定换方向了
- 他不知道你们俩今天的讨论结论

**每个人都有强助手，但产出跟一个人差不多。** 问题不在 AI 不够强，在于协作没解决。

### 我们做的是人的生意

我们的 business 最终是对人的——给真实的人创造真实的体验。

AI 再强，它不知道一杯咖啡的第一口什么感觉。它不知道音乐节现场的汗和风。它不知道一条文案在深夜刷到时是不是会让人停下来。

**这些感受，只有人有。这些判断，只有人能做。**

### 所以这套机器人解决两件事

**第一，让 AI 干更多的活，干得更好。** 拆任务、给角色、喂知识、设限制、互相质检——一句话触发一条流水线，AI 自己执行。

**第二，保存团队的判断力。**
- **一个问题就能问到答案，不需要问五遍。** 5 个 AI 角色从 5 个角度替你把问题想透了，你只需要在最后做判断。
- **一个人说一遍，其他人都知道。** 飞书群里实时可见，品牌知识写一次所有机器人自动调用，不需要重复传达。

### 回到开头

你不需要学会任何 AI 技术。因为技术会变，不会变的是：
- 你对「这个东西到底打不打动人」的判断
- 团队之间信息高效流转的需求
- 用最小的精力做出最好的决策的追求

**AI 越强，你的判断力越值钱——因为能干活的「人」（AI）变多了，但能做对决策的人没变多。**

## 附录

### 常见问题

**Q: 我不会写代码，能用吗？**
A: 部署需要基础的命令行操作（复制粘贴几条命令），日常使用完全不需要代码——在飞书上发消息就行。详见 [SETUP_GUIDE.md](SETUP_GUIDE.md)。

**Q: 要花多少钱？**
A: 项目本身免费。AI API 按量付费：DeepSeek 约 ¥0.01/次对话，一次完整脑暴约 ¥0.2-0.5。各平台新注册通常有免费额度。

**Q: 数据安全吗？**
A: 所有数据存在你自己的机器上，不会发到任何第三方。和 AI 服务商的通信走 HTTPS 加密。

**Q: 可以用 GPT/Claude 替代 DeepSeek 吗？**
A: 可以。改 `.env` 里的几个变量就行，不需要改代码。所有兼容 OpenAI 协议的模型都支持。

**Q: 机器人会断线吗？**
A: 所有机器人都有自动重连机制，网络恢复后自动恢复工作。
